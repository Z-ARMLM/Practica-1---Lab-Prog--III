Metadatos del documento

<!DOCTYPE>

Define el tipo de documento y la versión de HTML que se está utilizando.
Ejemplo: <!DOCTYPE html>
Debe ir al inicio del documento HTML, antes de la etiqueta <html>.

<html>

Representa la raíz del documento HTML, contiene todo el contenido de la página.
Ejemplo: <html lang="es"> ... </html>
Debe envolver todas las demás etiquetas (excepto <!DOCTYPE>).

<head>

Contenedor de metadatos y configuraciones del documento.
Ejemplo: <head><meta charset="UTF-8"></head>
No se muestra en la página, solo contiene datos de configuración.

<title>

Define el título del documento, mostrado en la barra del navegador o pestaña.
Ejemplo: <title>Mi página web</title>
Es importante para la accesibilidad y SEO.

<base>

Define la URL base para los enlaces relativos dentro del documento.
Ejemplo: <base href="https://ejemplo.com/">
Solo puede haber un <base> por documento.

<link>

Establece relación con recursos externos, como hojas de estilo.
Ejemplo: <link rel="stylesheet" href="estilos.css">
Se usa principalmente para cargar CSS o íconos.

<meta>

Define metadatos como charset, viewport o información para SEO.
Ejemplo: <meta name="description" content="Página de ejemplo">
No se muestra en la página, pero es esencial para buscadores y navegadores.

<style>

Contiene estilos CSS internos aplicados al documento.
Ejemplo: <style> body { background: lightblue; } </style>
Es mejor práctica usar CSS externo con <link>, pero puede usarse para estilos rápidos.

<script>

Permite incluir código JavaScript o enlazarlo desde un archivo externo.
Ejemplo: <script src="app.js"></script>
Puede colocarse en <head> o al final de <body> para mejorar el rendimiento.

Seccionamiento y estructura

<body>

Contiene todo el contenido visible de la página web.
Ejemplo: <body> <p>Hola mundo</p> </body>
Solo puede haber un <body> por documento.

<header>

Representa el encabezado de una página o sección.
Ejemplo: <header><h1>Título</h1></header>
Suele contener logo, navegación o títulos.

<footer>

Define el pie de página de una página o sección.
Ejemplo: <footer><p>© 2025 Mi sitio</p></footer>
Normalmente incluye información de contacto, derechos o enlaces legales.

<main>

Contiene el contenido principal del documento.
Ejemplo: <main><article>...</article></main>
Solo puede haber un <main> por documento.

<section>

Representa una sección temática dentro de un documento.
Ejemplo: <section><h2>Noticias</h2></section>
Ideal para dividir el contenido en bloques con un propósito específico.

<article>

Define contenido independiente y reutilizable (ejemplo: una noticia o post).
Ejemplo: <article><h2>Blog</h2></article>
Puede compartirse de manera autónoma fuera de la página principal.

<aside>

Representa contenido tangencial, como barras laterales o notas.
Ejemplo: <aside>Publicidad</aside>
No es parte del contenido principal, pero complementa la página.

<nav>

Define un bloque de enlaces de navegación.
Ejemplo: <nav><a href="#home">Inicio</a></nav>
Generalmente se usa para menús de navegación principales o secundarios.

<h1> ... <h6>

Encabezados de distintos niveles, del más importante (<h1>) al menos importante (<h6>).
Ejemplo: <h1>Título principal</h1>
Deben usarse jerárquicamente para mejorar accesibilidad y SEO.

<address>

Contiene información de contacto (persona, empresa, autor).
Ejemplo: <address>correo@ejemplo.com</address>
Suele colocarse en el <footer>.

Contenido en línea y texto

<p>

Define un párrafo de texto.
Ejemplo: <p>Este es un párrafo.</p>
Se separa automáticamente con márgenes en bloque.

<hr>

Representa un cambio temático en el contenido, usualmente mostrado como línea horizontal.
Ejemplo: <hr>
Se usa para separar secciones de manera visual.

<br>

Inserta un salto de línea dentro del texto.
Ejemplo: Hola<br>Mundo
No debe usarse en lugar de párrafos completos.

<pre>

Muestra texto preformateado, respetando espacios y saltos de línea.
Ejemplo: <pre>Texto con espacios</pre>
Útil para código o texto con formato fijo.

<blockquote>

Representa una cita en bloque de otra fuente.
Ejemplo: <blockquote>Cita larga</blockquote>
Generalmente se muestra con sangría.

<ol>

Define una lista ordenada con números o letras.
Ejemplo: <ol><li>Uno</li></ol>
El orden de los elementos es relevante.

<ul>

Define una lista no ordenada con viñetas.
Ejemplo: <ul><li>Elemento</li></ul>
Se usa cuando el orden no importa.

<li>

Define un elemento dentro de una lista (<ul> o <ol>).
Ejemplo: <li>Primer ítem</li>
Cada <li> representa un elemento de lista.

<dl>

Representa una lista de definiciones.
Ejemplo: <dl><dt>HTML</dt><dd>Lenguaje de marcado</dd></dl>
Útil para glosarios o descripciones.

<dt>

Define un término en una lista de definiciones.
Ejemplo: <dt>CSS</dt>
Se acompaña de <dd> que explica el término.

<dd>

Proporciona la definición de un término en <dl>.
Ejemplo: <dd>Hojas de estilo en cascada</dd>
Debe ir inmediatamente después de un <dt>.

<figure>

Contenedor para contenido gráfico o ilustrativo con su explicación.
Ejemplo: <figure><img src="img.png"></figure>
Puede contener una imagen, gráfico, diagrama, etc.

<figcaption>

Define un pie de figura asociado a <figure>.
Ejemplo: <figure><img src="img.png"><figcaption>Descripción</figcaption></figure>
Se usa para dar contexto o explicación a la figura.

<div>

Contenedor genérico de bloques.
Ejemplo: <div class="contenedor">Contenido</div>
No tiene significado semántico, se usa para agrupar y aplicar estilos.

<span>

Contenedor genérico en línea.
Ejemplo: <p>Texto <span class="rojo">resaltado</span></p>
Se usa para aplicar estilos o manipulación con JS a partes del texto.

Texto en línea semántico

<a>

Define un hipervínculo para navegar a otra página o recurso.
Ejemplo: <a href="https://ejemplo.com">Visitar</a>
Puede enlazar a sitios externos, internos o anclas en la misma página.

<em>

Representa énfasis en el texto, usualmente mostrado en cursiva.
Ejemplo: <em>Importante</em>
Aporta significado semántico de énfasis.

<strong>

Indica importancia en el texto, usualmente mostrado en negrita.
Ejemplo: <strong>Atención</strong>
Aporta significado semántico de relevancia.

<small>

Representa texto secundario o aclaratorio.
Ejemplo: <small>Nota al pie</small>
Se suele usar para advertencias o aclaraciones.

<cite>

Define el título de una obra citada (libro, película, etc.).
Ejemplo: <cite>El Quijote</cite>
Aporta contexto semántico sobre obras.

<q>

Representa una cita corta en línea.
Ejemplo: <q>Ser o no ser</q>
Generalmente los navegadores la muestran entre comillas.

<dfn>

Marca un término que está siendo definido en el texto.
Ejemplo: <dfn>HTML</dfn> es un lenguaje de marcado.
Útil para definiciones en documentos técnicos.

<abbr>

Representa una abreviatura o acrónimo.
Ejemplo: <abbr title="Cascading Style Sheets">CSS</abbr>
El atributo title explica su significado completo.

<time>

Representa una fecha u hora precisa.
Ejemplo: <time datetime="2025-09-16">16 de septiembre</time>
Útil para SEO y aplicaciones que procesan fechas.

<code>

Muestra un fragmento de código fuente.
Ejemplo: <code>console.log("Hola")</code>
Se representa en tipografía monoespaciada.

<var>

Representa una variable en una expresión matemática o en código.
Ejemplo: El área es <var>r</var>²π
Suele mostrarse en cursiva para diferenciarla del texto.

<samp>

Indica la salida de un programa o sistema.
Ejemplo: <samp>Error 404</samp>
Se muestra con fuente monoespaciada.

<kbd>

Representa la entrada de teclado.
Ejemplo: Presiona <kbd>Ctrl</kbd> + <kbd>C</kbd>
Se suele mostrar en estilo monoespaciado.

<sub>

Texto en subíndice.
Ejemplo: H<sub>2</sub>O
Se usa en fórmulas químicas o notas.

<sup>

Texto en superíndice.
Ejemplo: x<sup>2</sup>
Útil para exponentes o referencias.

<mark>

Resalta texto como si estuviera marcado con resaltador.
Ejemplo: <mark>Importante</mark>
Ideal para destacar palabras clave.

<b>

Aplica un estilo en negrita al texto, sin énfasis semántico.
Ejemplo: <b>Negrita</b>
Se usa para resaltar visualmente sin indicar importancia.

<i>

Aplica un estilo en cursiva o alternativo.
Ejemplo: <i>Palabra extranjera</i>
Se usa para nombres, pensamientos o términos técnicos.

<u>

Subraya texto con valor semántico (ejemplo: nombres propios).
Ejemplo: <u>Nombre propio</u>
Antes se usaba solo como estilo, ahora tiene un valor semántico específico.

<bdi>

Aísla un fragmento de texto que puede tener una dirección distinta.
Ejemplo: <bdi>عربي</bdi>
Útil en textos con escritura mixta (LTR y RTL).

<bdo>

Sobrescribe la dirección del texto.
Ejemplo: <bdo dir="rtl">Texto</bdo>
Se usa para controlar el flujo de escritura.

<ruby>

Contenedor para anotaciones en caracteres (tipografía asiática).
Ejemplo: <ruby>漢<rt>Kan</rt></ruby>
Usado en japonés, chino y otros idiomas.

<rt>

Texto de anotación dentro de <ruby>.
Ejemplo: <ruby>日<rt>ni</rt></ruby>
Muestra la pronunciación de un carácter.

<rp>

Texto alternativo para navegadores que no soportan <ruby>.
Ejemplo: <ruby>漢<rp>(</rp><rt>Kan</rt><rp>)</rp></ruby>
Normalmente muestra paréntesis alrededor de la anotación.

Multimedia e incrustación

<img>

Inserta imágenes en la página.
Ejemplo: <img src="imagen.jpg" alt="Descripción">

<picture>

Permite usar imágenes adaptables según el tamaño de pantalla.
Ejemplo: <picture> <source srcset="imagen-grande.jpg" media="(min-width:800px)"> <img src="imagen-peque.jpg" alt="Descripción"> </picture>

<source>

Define archivos multimedia dentro de <audio>, <video> o <picture>.
Ejemplo: <video controls><source src="video.mp4" type="video/mp4"></video>

<audio>

Reproduce archivos de audio.
Ejemplo: <audio controls><source src="audio.mp3" type="audio/mpeg"></audio>

<video>

Reproduce archivos de video.
Ejemplo: <video controls width="320"><source src="video.mp4" type="video/mp4"></video>

<track>

Proporciona subtítulos o descripciones a medios de audio o video.
Ejemplo: <track src="subtitulos.vtt" kind="subtitles" srclang="es" label="Español">

<iframe>

Incrusta otro documento web dentro de la página.
Ejemplo: <iframe src="pagina.html" width="600" height="400"></iframe>

<embed>

Incrusta contenido externo, como plugins o multimedia.
Ejemplo: <embed src="archivo.pdf" width="500" height="375">

<object>

Incrusta objetos, como documentos, multimedia o aplicaciones.
Ejemplo: <object data="archivo.pdf" width="500" height="375"></object>

<param>

Define parámetros para objetos incrustados con <object>.
Ejemplo: <object data="video.swf"><param name="autoplay" value="true"></object>

<canvas>

Permite dibujar gráficos y animaciones mediante JavaScript.
Ejemplo: <canvas id="miCanvas" width="400" height="200"></canvas>

<svg>

Crear gráficos vectoriales escalables directamente en HTML.
Ejemplo: <svg width="100" height="100"><circle cx="50" cy="50" r="40" fill="red"/></svg>

<math>

Permite mostrar fórmulas y expresiones matemáticas (MathML).
Ejemplo: <math><msup><mn>2</mn><mn>3</mn></msup></math>

Tablas

<table>

Define una tabla para organizar datos en filas y columnas.
Ejemplo:
<table>...</table>
Debe contener filas (<tr>) y celdas (<td> o <th>).

<caption>

Agrega un título descriptivo a la tabla.
Ejemplo:
<caption>Lista de Productos</caption>
Se coloca inmediatamente después de la etiqueta <table>.

<colgroup>

Define un grupo de columnas dentro de una tabla para aplicar estilos.
Ejemplo:
<colgroup><col span="2" style="background:lightgray"></colgroup>
Se coloca antes de <thead>.

<col>

Especifica propiedades de una columna o grupo de columnas.
Ejemplo:
<col style="width:50%">
Se usa dentro de <colgroup>.

<thead>

Encabezado de la tabla, contiene las filas con títulos de columna.
Ejemplo:
<thead><tr><th>Nombre</th></tr></thead>
Se coloca antes de <tbody>.

<tbody>

Cuerpo de la tabla, contiene las filas de datos.
Ejemplo:
<tbody><tr><td>Juan</td></tr></tbody>
Puede usarse varias veces en una tabla grande.

<tfoot>

Pie de tabla, contiene filas con información resumen.
Ejemplo:
<tfoot><tr><td>Total</td></tr></tfoot>
Se coloca después de <tbody>.

<tr>

Define una fila en una tabla.
Ejemplo:
<tr><td>Dato1</td></tr>
Se utiliza dentro de <thead>, <tbody> o <tfoot>.

<th>

Celda de encabezado en una tabla.
Ejemplo:
<th>Nombre</th>
Por defecto aparece en negrita y centrado.

<td>

Celda de datos en una tabla.
Ejemplo:
<td>Juan</td>
Se coloca dentro de una fila (<tr>).

Formularios

<form>

Define un formulario interactivo para recopilar datos del usuario.
Ejemplo:
<form action="procesar.php" method="post">...</form>
Puede contener inputs, botones y otros elementos de entrada.

<input>

Campo de entrada de datos en un formulario.
Ejemplo:
<input type="text" name="usuario">
Puede tener muchos tipos: text, password, checkbox, radio, etc.

<textarea>

Área de texto de varias líneas para ingresar contenido.
Ejemplo:
<textarea name="mensaje">Escribe aquí...</textarea>
Útil para comentarios o mensajes largos.

<button>

Botón en un formulario o página.
Ejemplo:
<button type="submit">Enviar</button>
Puede tener los tipos: submit, reset o button.

<select>

Lista desplegable para elegir una opción.
Ejemplo:
<select name="pais"><option>México</option></select>
Puede contener <option> y <optgroup>.

<option>

Opción dentro de un menú desplegable (<select>).
Ejemplo:
<option value="mx">México</option>
Puede marcarse como seleccionada con el atributo selected.

<optgroup>

Grupo de opciones dentro de un <select>.
Ejemplo:
<optgroup label="América"><option>Chile</option></optgroup>
Se usa para organizar listas extensas.

<label>

Etiqueta para un campo de formulario.
Ejemplo:
<label for="usuario">Nombre:</label>
Asociar con el atributo for mejora la accesibilidad.

<fieldset>

Agrupa campos relacionados dentro de un formulario.
Ejemplo:
<fieldset>...</fieldset>
Se suele acompañar con <legend>.

<legend>

Define un título o leyenda para un grupo de campos (<fieldset>).
Ejemplo:
<legend>Información personal</legend>
Mejora la organización y accesibilidad del formulario.

<datalist>

Lista de valores sugeridos para un campo de entrada (<input>).
Ejemplo:
<input list="navegadores"> <datalist id="navegadores"> <option value="Chrome"> </datalist>
Permite autocompletar opciones.

<output>

Muestra el resultado de un cálculo en un formulario.
Ejemplo:
<output name="resultado">10</output>
Puede actualizarse dinámicamente con JavaScript.

<progress>

Barra de progreso que muestra el avance de una tarea.
Ejemplo:
<progress value="70" max="100"></progress>
Útil para cargas o descargas.

<meter>

Representa un valor escalar en un rango conocido.
Ejemplo:
<meter value="0.7" min="0" max="1">70%</meter>
Se usa para valores medibles (ej. puntuación, nivel).

Scripting y APIs

<noscript>

Contenido alternativo que se muestra si JavaScript está deshabilitado en el navegador.
Ejemplo:
<noscript>Por favor activa JavaScript para usar esta página.</noscript>
Útil para accesibilidad y soporte en navegadores antiguos.

<template>

Define un fragmento de contenido HTML reutilizable que no se renderiza al cargar la página.
Ejemplo:
<template id="card"> <div class="item">...</div> </template>
Se inserta dinámicamente con JavaScript.

<slot>

Punto de inserción para contenido en Web Components.
Ejemplo:
<slot name="titulo">Título por defecto</slot>
Permite personalizar contenido dentro de componentes reutilizables.

Etiquetas obsoletas / deprecadas

<acronym>

Definía acrónimos.
Ejemplo: <acronym title="World Health Organization">WHO</acronym>
Reemplazada por <abbr>.

<applet>

Insertaba applets de Java.
Ejemplo: <applet code="miApplet.class"></applet>
Reemplazada por <object>.

<big>

Mostraba texto más grande.
Ejemplo: <big>Texto grande</big>
Reemplazado por CSS (font-size).

<center>

Centraba el contenido.
Ejemplo: <center>Texto centrado</center>
Reemplazado por CSS (text-align: center).

<dir>

Lista de directorios.
Ejemplo: <dir><li>Archivo</li></dir>
Reemplazado por <ul>.

<font>

Definía fuente, color y tamaño de texto.
Ejemplo: <font color="red">Texto</font>
Reemplazado por CSS.

<frame>

Definía un marco dentro de un frameset.
Ejemplo: <frame src="pagina.html">
Obsoleto, reemplazado por <iframe>.

<frameset>

Definía un conjunto de marcos.
Ejemplo: <frameset cols="50%,50%">...</frameset>
Reemplazado por <iframe>.

<noframes>

Contenido alternativo si el navegador no soportaba frames.
Ejemplo: <noframes>Tu navegador no soporta frames</noframes>
Ya no tiene sentido en HTML5.

<strike>

Texto tachado.
Ejemplo: <strike>Texto</strike>
Reemplazado por <s> o CSS (text-decoration).

<tt>

Texto monoespaciado (teletipo).
Ejemplo: <tt>Código</tt>
Reemplazado por <code> o CSS.

<u> (versión antigua)

Subrayaba texto sin valor semántico.
Ejemplo: <u>Subrayado</u>
Ahora tiene un nuevo uso semántico en HTML5.

<xmp>

Mostraba texto preformateado.
Ejemplo: <xmp>Código</xmp>
Reemplazado por <pre><code>.

<marquee>

Texto en movimiento (desplazamiento automático).
Ejemplo: <marquee>Texto en movimiento</marquee>
Nunca fue estándar, obsoleto.

<blink>

Texto parpadeante (solo Netscape).
Ejemplo: <blink>Texto</blink>
Nunca fue estándar, totalmente obsoleto.

<isindex>

Campo de búsqueda en documento.
Ejemplo: <isindex prompt="Buscar">
Obsoleto, ya lo cubre <form> + <input>.

<rb>

Parte del modelo Ruby en versiones antiguas.
Ejemplo: <rb>漢</rb>
Obsoleto, reemplazado en el modelo moderno de Ruby.
